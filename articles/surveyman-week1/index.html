Week 1 - Surveyman

April 7, 2015 in react surveyman

Last week, I announced that I have applied for GSOC project called Surveyman as a part of which I’ll be building a React application that allows users to build a survey.

How will this be different from the countless other similar tools that exist? For the long answer, I’d advise you to read the paper but the short answer is that SurveyMan exposes a lightweight Domain Specific Language for designing and deploying surveys. It then does static and dynamic analysis on the survey and find survey bugs and control the quality of responses.

My project basically deals with designing a front-end single-page application which makes it easy for non-programmers to use the DSL and build out the survey. The most fun part about the project is that the application needs to be in React and this would be great opportunity to build something useful and learn React at the same time.

Before the GSOC deadline period ended, I had setup an application that supported drag drop from the Toolbox (shown above) to the Dropzone. To keep the demo simple, all widgets when dropped were added to the end of the survey e.g. When a Question widget is dropped onto the page, it gets added to the end of the Survey. This was natural extension to the Starter task outlined by the mentor.

During this period, I spent most of time learning React and experimenting with drag and drop APIs as a part of which I built a simple drag-and-drop shopping cart example.

After the proposal was submitted and the wireframe finalized, my main goal was to set up an overlying architecture of how the application will be organized. Another important part was to figure what key libraries the project will depend on. In the initial version, all events were setup at the root controller which was responsible for updating the state and passing down the changes to the sub components. This week I migrated the application to support the Flux architecture by using the Reflux library to dispatch actions to stores.

Other than that I also included the extremely useful drag and drop library - ReactDND which makes it very easy to setup drag sources and drop targets. Rather than having one drop zone where all widgets are dropped I extended the functionality of all survey items to act as drop targets and accordingly action the event.

The final piece of the puzzle includes setting up a CommonJS module system for which I used browserify. Since most of the online react example were based on ES6 I decided to jump the gun and start using ES6 myself. This was easy to setup via the babelify npm module. For more info, you can dive into the package.json and see the configuration for yourself.

The application as it currently stands can be seen here and the code can be browsed on Github.

My plan for the next week is to improve the UI for taking the input, which for the time being, is a simple JS prompt. The next step would be to enable other CRUD features namely, update and delete for survey components.

Till next time!
